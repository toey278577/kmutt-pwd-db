## 2. การเริ่มต้นใช้งาน

### ความต้องการของระบบ
- Node.js v18+
- PostgreSQL ที่รันอยู่
- Database ชื่อ `kmutt_pwd_db`

### ขั้นตอนการรัน

**ขั้นที่ 1 — รัน Backend**
```bash
cd backend
npm run dev
```
จะเห็นข้อความ: `Server running on port 5000`

**ขั้นที่ 2 — รัน Frontend** (เปิด terminal ใหม่)
```bash
cd frontend
npm run dev
```
จะเห็นข้อความ: `Local: http://localhost:5173/`

**ขั้นที่ 3 — Seed ข้อมูลเริ่มต้น** (ครั้งแรกเท่านั้น)
```bash
cd backend
node prisma/seed.js
```
จะสร้าง:
- ประเภทความพิการ 7 ประเภท
- บัญชี Admin เริ่มต้น: `admin@kmutt.ac.th` / `admin123`

**ขั้นที่ 4 — เข้าใช้งาน**

เปิด browser ไปที่ `http://localhost:5173` แล้ว login ด้วย Admin account

## 3. ระบบเข้าสู่ระบบ (Login)

### วิธีเข้าสู่ระบบ
1. เปิด browser ไปที่ `http://localhost:5173`
2. ระบบจะ redirect ไปหน้า Login อัตโนมัติ
3. กรอก **อีเมล** และ **รหัสผ่าน**
4. กดปุ่ม **เข้าสู่ระบบ**

### บัญชีเริ่มต้น (Admin)
| ฟิลด์ | ค่า |
|-------|-----|
| อีเมล | admin@kmutt.ac.th |
| รหัสผ่าน | admin123 |

### การออกจากระบบ (Logout)
- กดไอคอน **ออกจากระบบ** (รูปลูกศรออก) ที่มุมล่างซ้ายของ sidebar
- ระบบจะ redirect กลับหน้า Login

### หมายเหตุ
- Token หมดอายุ **8 ชั่วโมง** หลังจาก login (1 วันทำงาน)
- ถ้า token หมดอายุ ระบบจะ redirect กลับหน้า Login อัตโนมัติ
- ไม่มีระบบสมัครสมาชิก — Admin เป็นผู้สร้างบัญชีให้เจ้าหน้าที่

## 5. หน้าข้อมูลคนพิการ

**เข้าถึง:** กดเมนู **ข้อมูลคนพิการ** ในแถบด้านซ้าย

### ฟังก์ชันหลัก

#### ค้นหา
- พิมพ์ชื่อหรือเลขบัตรประชาชน แล้วกด **ค้นหา** หรือกด Enter
- กด **ล้าง** เพื่อแสดงทุกรายชื่อ

#### เพิ่มคนพิการใหม่ *(Admin / Staff เท่านั้น)*
1. กดปุ่ม **เพิ่มคนพิการ** (ปุ่มส้มมุมขวาบน)
2. กรอกข้อมูลในหน้าต่าง popup:

**ข้อมูลส่วนตัว (จำเป็น)**
| ฟิลด์ | หมายเหตุ |
|-------|---------|
| ชื่อ-นามสกุล * | ต้องกรอก |
| เลขบัตรประชาชน | 13 หลัก (unique) |
| เพศ * | ชาย / หญิง / อื่นๆ |
| วันเกิด | รูปแบบ YYYY-MM-DD |
| สัญชาติ | ค่าเริ่มต้น: ไทย |
| ศาสนา | ค่าเริ่มต้น: พุทธ |
| สถานภาพ | โสด / สมรส / อื่นๆ |
| ระดับการศึกษา | ข้อความอิสระ |
| สถานะชีวิต | มีชีวิต / เสียชีวิต |

**ข้อมูลติดต่อ**
| ฟิลด์ | หมายเหตุ |
|-------|---------|
| เบอร์โทร | ตัวเลข |
| Email | รูปแบบ email |
| จังหวัด | ข้อความอิสระ |
| ที่อยู่ | ข้อความยาว |

3. กดปุ่ม **บันทึก**

#### แก้ไขข้อมูล *(Admin / Staff เท่านั้น)*
- กดไอคอน **ดินสอ** ในแถวนั้น → แก้ไข → กด **บันทึก**

#### ลบข้อมูล *(Admin / Staff เท่านั้น)*
- กดไอคอน **ถังขยะ** → ยืนยัน → ลบ
- **หมายเหตุ:** การลบคนพิการจะลบข้อมูลที่เกี่ยวข้องทั้งหมด (อบรม, ทำงาน, ติดตามผล, ทักษะ)

#### ดูรายละเอียด
- กดไอคอน **ตา** → ไปหน้ารายละเอียดคนพิการ

## 6. หน้ารายละเอียดคนพิการ

**เข้าถึง:** กดไอคอนตา ในหน้าข้อมูลคนพิการ

### ข้อมูลหัว
- **ชื่อ-นามสกุล** และ **อายุ** (คำนวณจากวันเกิดอัตโนมัติ)
- เพศ, จังหวัด
- สถานะชีวิต (มีชีวิต / เสียชีวิต)

### แท็บทั้ง 6

#### แท็บ 1: ข้อมูลพื้นฐาน
- ข้อมูลส่วนตัวทั้งหมด + วันเกิดพร้อมอายุ
- ข้อมูลความพิการ (ประเภท, ระดับ, อุปกรณ์, ข้อจำกัด, ความต้องการ)

#### แท็บ 2: การอบรม *(เพิ่ม/ลบ: Admin/Staff)*
กดปุ่ม **เพิ่ม** แล้วกรอก:
| ฟิลด์ | หมายเหตุ |
|-------|---------|
| ชื่อหลักสูตร * | ต้องกรอก |
| หน่วยงาน | ผู้จัดอบรม |
| วันเริ่ม / วันสิ้นสุด | รูปแบบวันที่ |
| รูปแบบ | อบรม (Train) / เรียนรู้ (Learn) / ฝึกงาน (Earn) |
| ทักษะที่ได้รับ | รายการทักษะที่เรียนในหลักสูตรนี้ |
| ผลประเมิน | ผ่าน / ไม่ผ่าน ฯลฯ |

#### แท็บ 3: ประสบการณ์งาน *(เพิ่ม/ลบ: Admin/Staff)*
| ฟิลด์ | หมายเหตุ |
|-------|---------|
| ชื่อองค์กร * | ต้องกรอก |
| ประเภทงาน | ชื่อตำแหน่ง/งาน |
| รูปแบบงาน | ฝึกงาน / จ้างงาน / Freelance |
| รายได้ | หน่วย: บาท |
| วันเริ่ม / วันสิ้นสุด | ว่างไว้ = ปัจจุบัน |
| ผลลัพธ์ / ความสำเร็จ | บรรยายผลการทำงาน |

#### แท็บ 4: ทักษะ *(เพิ่ม/ลบ: Admin/Staff)*
| ฟิลด์ | ตัวเลือก |
|-------|---------|
| ชื่อทักษะ * | ข้อความอิสระ (เช่น Python, Excel) |
| ระดับ * | Beginner (เริ่มต้น) / Intermediate (ปานกลาง) / Advanced (เชี่ยวชาญ) |

- ทักษะแสดงเป็น **tag สี** ตามระดับ: เหลือง (เริ่มต้น), ฟ้า (ปานกลาง), เขียว (เชี่ยวชาญ)

#### แท็บ 5: ติดตามผล *(เพิ่ม/ลบ: Admin/Staff)*
| ฟิลด์ | หมายเหตุ |
|-------|---------|
| วันที่ติดตาม * | ต้องกรอก |
| สถานะงาน * | มีงานทำ / ว่างงาน / ศึกษาต่อ |
| ประเภทงาน | ชื่อตำแหน่ง/งาน |
| รายได้ | หน่วย: บาท |
| ทักษะตรงสาย | ตรงสาย / ไม่ตรงสาย |
| ความพึงพอใจ | ข้อความอิสระ |
| ปัญหา / ข้อเสนอแนะ | บรรยายปัญหาที่พบ |

#### แท็บ 6: สถานประกอบการ *(เพิ่ม/ลบ: Admin/Staff)*
เชื่อมโยงคนพิการกับสถานประกอบการที่บันทึกไว้
| ฟิลด์ | หมายเหตุ |
|-------|---------|
| สถานประกอบการ * | เลือกจากรายการที่มีในระบบ |
| บทบาท * | ฝึกงาน / จ้างงาน / อื่นๆ |
| วันเริ่ม / วันสิ้นสุด | ช่วงเวลา |
| เงินสนับสนุน | หน่วย: บาท |
| รายละเอียดการสนับสนุน | บรรยายเพิ่มเติม |
| หมายเหตุ | ข้อความเพิ่มเติม |

## 7. หน้าสถานประกอบการ

**เข้าถึง:** กดเมนู **สถานประกอบการ** ในแถบด้านซ้าย

### ฟังก์ชัน

#### เพิ่มสถานประกอบการ *(Admin / Staff เท่านั้น)*
1. กดปุ่ม **เพิ่มองค์กร**
2. กรอกข้อมูล:

| ฟิลด์ | หมายเหตุ |
|-------|---------|
| ชื่อองค์กร * | ต้องกรอก |
| ประเภทธุรกิจ | เช่น IT, การผลิต, ค้าปลีก |
| ที่อยู่ | ที่อยู่เต็ม |
| ผู้ติดต่อ | ชื่อผู้ดูแล |
| เบอร์โทร | ตัวเลข |
| Email | รูปแบบ email |
| หมายเหตุ | ข้อความเพิ่มเติม |

3. กด **บันทึก**

#### แก้ไข / ลบ *(Admin / Staff เท่านั้น)*
- กดไอคอน **ดินสอ** หรือ **ถังขยะ** ในแถวนั้น
- **หมายเหตุ:** การลบองค์กรจะลบความสัมพันธ์กับคนพิการทั้งหมดในแท็บ "สถานประกอบการ" โดยอัตโนมัติ

## 8. หน้าติดตามผล

**เข้าถึง:** กดเมนู **ติดตามผล** ในแถบด้านซ้าย

แสดงรายการติดตามผล **รวมทุกคน** เรียงจากวันที่ล่าสุด

### คอลัมน์ที่แสดง
| คอลัมน์ | ความหมาย |
|---------|----------|
| วันที่ติดตาม | วันที่บันทึกข้อมูล |
| ชื่อ | กดได้เพื่อไปหน้ารายละเอียดคนพิการ |
| สถานะงาน | มีงานทำ / ว่างงาน / ศึกษาต่อ |
| ประเภทงาน | ตำแหน่ง/งานที่ทำ |
| รายได้ | รายได้ต่อเดือน |
| ทักษะสอดคล้อง | ทักษะตรงสาย / ไม่ตรงสาย |
| ความพึงพอใจ | ความพึงพอใจในงาน |
| ปัญหา | ปัญหาที่พบในการทำงาน |

> การเพิ่ม/ลบข้อมูลติดตามผล ทำได้ใน **แท็บ "ติดตามผล"** ของหน้ารายละเอียดคนพิการแต่ละคน

## 9. หน้าจัดการผู้ใช้งาน

**เข้าถึง:** กดเมนู **จัดการผู้ใช้** ในแถบด้านซ้าย *(เห็นเฉพาะ Admin)*

### ฟังก์ชัน

#### เพิ่มผู้ใช้ใหม่
1. กดปุ่ม **เพิ่มผู้ใช้**
2. กรอกข้อมูล:

| ฟิลด์ | หมายเหตุ |
|-------|---------|
| ชื่อ-นามสกุล * | ชื่อที่แสดงใน sidebar |
| อีเมล * | ใช้เป็น username สำหรับ login |
| รหัสผ่าน * | กำหนดโดย Admin |
| บทบาท | Admin / Staff / Viewer |

3. กด **เพิ่มผู้ใช้** — Admin ต้องแจ้ง email และ password ให้ผู้ใช้เอง

#### แก้ไขผู้ใช้
- กดไอคอน **ดินสอ** → แก้ไขชื่อ, email, บทบาท หรือเปลี่ยน password ได้
- ถ้าเว้นช่อง password ว่างไว้ รหัสผ่านจะไม่เปลี่ยน

#### เปิด/ปิดการใช้งาน
- กดที่ **badge สถานะ** (ใช้งาน / ปิดใช้งาน) ในแถวนั้น
- บัญชีที่ปิดใช้งาน จะ login ไม่ได้

#### ลบผู้ใช้
- กดไอคอน **ถังขยะ** → ยืนยัน
- ไม่สามารถลบบัญชีของตัวเองได้

## 11. สรุปบัคที่พบและแก้ไขแล้ว

### บัค Backend (รอบแรก)
| บัค | สาเหตุ | การแก้ไข |
|-----|--------|---------|
| POST person ล้มเหลว (DateTime error) | Prisma ปฏิเสธ string `"2024-01-01"` | เพิ่มฟังก์ชัน `toDate()` แปลงเป็น `new Date()` |
| Enum field crash ตอนส่งค่าว่าง | `""` ไม่ใช่ Enum value ที่ถูกต้อง | เพิ่มฟังก์ชัน `toEnum()` แปลง `""` เป็น `null` |
| Income บันทึกผิด (string vs Decimal) | `<input type="number">` ส่งค่าเป็น string | เพิ่มฟังก์ชัน `toNum()` ใช้ `parseFloat()` |
| FollowUpList โหลดช้า | ใช้ `for...of + await` แบบ sequential | เปลี่ยนเป็น `Promise.all()` |

### บัค Frontend (รอบแรก)
| บัค | สาเหตุ | การแก้ไข |
|-----|--------|---------|
| Modal มีพื้นหลังโปร่งแสง | ไม่ได้กำหนด `bg-white` ใน modal-box | เพิ่ม `bg-white` |
| Dropdown option ข้อความซ้อน | DaisyUI select ไม่รองรับ custom style | เปลี่ยนเป็น custom Tailwind select + ChevronDown |

### บัค ระบบ Login (รอบสอง)
| บัค | สาเหตุ | การแก้ไข |
|-----|--------|---------|
| Login ไม่ได้หลัง restart | Admin user ไม่ถูก seed ลง DB ที่ถูก | รัน `node prisma/seed.js` ใหม่ |

| Search input ดูไม่สอดคล้องกับ UI อื่น | ใช้ DaisyUI `input-bordered` class | เปลี่ยนเป็น custom Tailwind style |
| FollowUpList ขาดคอลัมน์ใหม่ | satisfaction + issues ไม่แสดง | เพิ่มคอลัมน์ + กด navigate ไปหน้ารายละเอียด |
